语言(Languages)

  1. 静态编译型(OOP):C#/ Java/ C++ /C /golang
  2. 动态解释型(OOP):PHP/Ruby/Python/Javascript
  3. 函数编程(FP)：F#/Scala/Haskell/Erlang
    学习建议：
  • 如果大家以后要从事企业应用系统(ERP,CRM,OA,BI,银行，电信的业务系统等)的开发，主流的平台就是.net 与 java 大家可以选择这两者好好学习;
  • 如果大家以后想去大型互联网公司(qq,taobao 等)参与他们平台的一些功能的开发或主要开发web应用系统(SNS,门户站点==)，大家可以主要学习:java PHP Ruby Python,golang,nodejs 这些的其中一两种;
  • 如果大家以后要想专做web前端技术的开发，就要认真学好:javascript;
  • 如果大家以后要向系统架构，高性能，服务端系统开发或分布式，并行计算方面发展，可以主要学习:.net，java ，golang， nodejs及函数编程语言;
  • 如果大家像以后从事移动设备应用程序开发，可以主要学习：java(Android) C#(WP7) C++(其他平台手机）;

平台(Platforms)

  • JavaEE:主要用于开发基于java语言的企业应用系统，B/S结构的服务端，高性能，分布式系统的开发;
  • .NET:主要用于开发基于Windows .net平台语言(C#,VB,C++)的企业应用系统,B/S结构的服务端，高性能，分布式系统的开发;
  • LAMP:开源企业级系统开发平台,主要利用常用开源系统（Linux,Apache,Mysql,PhP,Perl,Python,Ruby on rails)开发企业应用系统，B/S结构的系统开发，对高性能与分布式系统，通常需要研究其他第三方的开源组件;
  • 智能手机平台：Android,iphone(Ios),Html5现今三大主流的手机OS平台
  • 游戏开发平台：主要是两大块，手机游戏，windows游戏，现在主要还是用C++，java 一般来说，选择了什么语言就选择了什么开发平台，或者说，选择哪个开发平台，基于也就确实你要学习或掌握什么语言，大家可以看着办

设计(Design)

  • OOA&D
  • 设计模式(Design Patterns)
  • 重构(Refactoring)
  • 领域驱动设计(DDD)
  • 对象设计：角色、责任和协作(RDD)
  • 敏捷软件开发(Agile/XP/Scrum)
  • 代码之美
  • 架构模式
  • 分析模式

框架(Framewroks)

  • 企业应用系统开发:Hibernate,Spring,Ibaist,ActiveRecord,MVC框架，Ms.Entlib等
  • Web系统开发：PHP框架，Ruby On rails,Python Django 等
  • 手机开发框架：手机OS都有自己开发框架

架构(Architectures)

  • 企业级应用程序架构模式
  • 使用Microsoft.NET的企业解决方案模式
  • 面向模式的软件架构
  • 分布式架构设计

领域专家或大师或升天(Expert/Master/GameOver)

  • 在不断的学习与工作中，最好专门从事某一领域（Web前端技术，电子商务，移动应用，搜索引擎，分布式计算等）工作，这样有可能在这个领域内做得精，强。如果没有这样的机遇与能力，到最后要么早就转行了，要么安心做一名长久的开发人员（其实这也不错，只是大部分企业不能让你有技术上的积累）

其他一些必备工具与技能(Skills)

  • 平时有空经常温故一些基础知识：数据结构，算法，操作系统，数据库等
  • 源代码管理工具：git
  • 开发工具使用技巧，常用软件测试技术
  • 理解软件开发方法，软件工程及软件质量管理
  • 常研究一些好的开源项目
  • 关注的行业的发展信息
  • 可以用用linux 以上是对开发人员学习路线的一些建议，每个职业规划不一样，我只是给了一些参考，每个人不一定要成为专家或大师，但是可以努力成一个专业且优秀的开发人员。
